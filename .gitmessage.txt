# 설명 (선택) - 무엇을 왜 수정했는지
#
# 예시:
# feat: 퀘스트 추가 화면 UI 구현
# fix: XP가 저장되지 않는 버그 수정
# refactor: 퀘스트 카드 로직 리팩토링
# chore: tailwind 설정 파일 정리
#
# Types:
#   feat      - 새로운 기능
#   fix       - 버그 수정
#   refactor  - 리팩토링 (기능 변화 없음)
#   style     - UI 스타일, CSS 등
#   chore     - 기타 설정, 환경 작업
#   docs      - 문서 작성 또는 수정
#
# 주의: 제목은 50자 이내, 첫 글자는 소문자, 마침표 없이 작성!
